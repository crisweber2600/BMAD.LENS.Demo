# Documentation Quality Standards

> **Reference Exemplar:** `docs/lens-sync/OldNorthStar/` in the control repo.
> Every generated document MUST match the depth, detail, and style of those reference files.

---

## Quality Gates

### Minimum Line Counts (per document)

| Document | Minimum Lines | Target Lines |
|----------|--------------|--------------|
| architecture.md | 200 | 300+ |
| api-surface.md | 150 | 250+ |
| data-model.md | 200 | 400+ |
| integration-map.md | 150 | 300+ |
| onboarding.md | 200 | 400+ |
| migration-map.md | 150 | 350+ |

If a generated document falls below the minimum, the agent MUST re-read source files and expand content before accepting the output.

### Required Content Types

Every document MUST contain:

1. **Real code examples** — extracted from the actual codebase, not invented
2. **Architecture diagrams** — ASCII art box diagrams using `┌─┐│└─┘` characters
3. **Detailed tables** — with real data, not placeholders
4. **Cross-references** — links to all related docs in the set
5. **BMAD header** — standard metadata header with generator, system, status

### CRITICAL: Source File Reading Requirement

**Before generating any document, the agent MUST:**

1. Read actual source files (`read_file`) — entity classes, controllers, config files
2. Extract real class definitions, method signatures, and configuration blocks
3. Count actual endpoints, entities, and integrations from source code
4. Include verbatim code snippets from the repository (with attribution)

**NEVER generate placeholder content like:**
- `{service_name}` or `{framework}`
- `{for dep in dependencies}` loops
- Invented class names or endpoint paths
- Generic descriptions not based on actual code inspection

### Standard BMAD Header

Every generated document MUST start with:

```markdown
# {Actual Service Name} - {Document Type}

> **Generated**: SCOUT Analyze Codebase + Generate Docs Workflow
> **System**: LENS Discovery System
> **Status**: BMAD-Ready Documentation

---
```

### Standard Footer

Every generated document MUST end with:

```markdown
---

## Related Documentation

- [Architecture](architecture.md) - System architecture overview
- [API Surface](api-surface.md) - API endpoint inventory
- [Data Model](data-model.md) - Entity documentation
- [Integration Map](integration-map.md) - System integration points
- [Migration Map](migration-map.md) - Modernization pathway
- [Onboarding Guide](onboarding.md) - Developer onboarding

---

*Generated by LENS System - SCOUT Discovery Workflow*
```

### Six Required Documents

Every target MUST produce all six documents:

1. `architecture.md` — System architecture, tech stack, patterns, deployment
2. `api-surface.md` — Every API endpoint with methods, paths, auth details
3. `data-model.md` — Every entity with fields, relationships, code examples
4. `integration-map.md` — All internal/external integrations with diagrams
5. `migration-map.md` — Technology migration strategy (legacy→modern)
6. `onboarding.md` — Developer quick-start with actual setup commands

### Document Completeness Checklist

Before accepting a document set as complete, verify:

- [ ] All 6 documents generated
- [ ] Each document meets minimum line count
- [ ] Each document contains real code examples from the repo
- [ ] Each document has ASCII art diagrams (not just mermaid)
- [ ] Each document has the standard BMAD header and footer
- [ ] Cross-references link to all sibling documents
- [ ] No placeholder variables (`{var}`) remain in output
- [ ] Tables contain actual data, not template markers
